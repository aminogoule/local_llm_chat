# 🌙 imsoft chat

Лёгкий одностраничный веб-клиент для общения с **локальной LLM** через сервер
[LMStudio](https://lmstudio.ai/) по его OpenAI-совместимому HTTP API.

Без сборки, без зависимостей, без фреймворков — всего три статических файла:
`index.html`, `style.css`, `app.js`. Интерфейс на русском языке.

## Возможности

- 💬 **Чат с локальной моделью** через `POST /v1/chat/completions`.
- ⚡ **Потоковый вывод** (SSE) с возможностью остановки генерации.
- 🗂 **Несколько диалогов** с историей в боковой панели.
- ✏️ **Управление чатами** — переименование и удаление через контекстное меню
  (кнопка ⋯ или двойной клик по названию).
- 💾 **Сохранение в `localStorage`** — чаты, активный диалог и настройки
  переживают перезагрузку страницы.
- ⚙️ **Параметры генерации** — выбор модели, системный промпт, температура,
  лимит токенов, переключение потокового режима.
- 🎨 **Тёмная и светлая темы** (тёмная по умолчанию).
- 📝 **Безопасный markdown-рендер** ответов — собственный, без зависимостей,
  с экранированием HTML и подсветкой блоков кода с кнопкой «копировать».
- 🟢 **Индикатор статуса сервера** с автоматическим опросом каждые 30 секунд.
- 📱 **Адаптивный интерфейс** — боковая панель-оверлей и тач-доступные действия
  на мобильных устройствах.

## Быстрый старт

Собирать или устанавливать ничего не нужно. Достаточно поднять любой статический
сервер из каталога проекта:

```sh
python3 -m http.server 8080
# затем откройте http://localhost:8080
```

Перед использованием убедитесь, что **сервер LMStudio запущен и в него загружена
модель** (по умолчанию слушает `http://localhost:1234`).

### Настройка адреса API

Адрес сервера задаётся константой `API_BASE` в начале [app.js](app.js):

```js
const API_BASE = "";
```

- **Продакшен.** Пустая строка — относительный путь. Приложение обслуживается за
  Apache на `chat.imsoft.pro`, который проксирует `/v1/...` на LMStudio.
- **Локальная разработка.** У обычного статического сервера нет эндпоинта `/v1`,
  поэтому при пустом `API_BASE` все запросы упадут. Укажите адрес LMStudio
  напрямую, например:

  ```js
  const API_BASE = "http://192.168.1.57:1234";
  ```

Используемые эндпоинты LMStudio:

| Эндпоинт                   | Назначение                                  |
| -------------------------- | ------------------------------------------- |
| `GET /v1/models`           | список моделей и проверка доступности сервера |
| `POST /v1/chat/completions`| генерация ответа (потоковая и обычная)       |

## Архитектура (кратко)

`app.js` — единый IIFE в строгом режиме. Ключевые моменты:

- **Состояние и хранение.** Всё состояние в одном объекте `state`
  (`chats`, `activeId`, `settings`), зеркалируется в `localStorage` под ключами
  `imsoft.*`. Тема хранится отдельно (`imsoft.theme`).
- **Markdown-рендер.** Сначала экранируется весь HTML, затем заново вводится
  фиксированный белый список элементов; блоки кода вырезаются в плейсхолдеры до
  инлайн-разбора. Порядок «сначала экранирование» важен — вывод модели
  недоверенный и вставляется через `innerHTML`.
- **Стриминг.** `ReadableStream` читается вручную, буферизуется и парсится
  построчно как SSE-события `data:`. Остановку обслуживает единственный
  `AbortController`.
- **Рендеринг.** Без виртуального DOM: секции перестраиваются из `state`, а
  текущий ответ ассистента дополняется напрямую по мере прихода дельт.

Подробнее — в [CLAUDE.md](CLAUDE.md).
